Weather Aggregator

One human-friendly query → five public weather APIs in parallel.

How to use

  1. Enter a city and country (required).
  2. Optionally pin the state (helpful for ambiguous names like “Springfield”), pick a date, choose units.
  3. Click Get weather. The page will geocode your input, then fan out to five providers in parallel and show you the results plus timing proof.
Use this service from the terminal or Postman

The endpoint is GET /weather with query params city, country (required), state, date (YYYY-MM-DD), units (celsius | fahrenheit).

curl

curl 'http://127.0.0.1:8000/weather?city=Berlin&country=DE'

curl with diacritics

curl -G 'http://127.0.0.1:8000/weather' \
  --data-urlencode 'city=São Paulo' \
  --data-urlencode 'country=BR'

Postman

  1. Create a new GET request.
  2. URL: http://127.0.0.1:8000/weather
  3. Open the Params tab and add: city=Berlin, country=DE, optionally state, date, units.
  4. Send. The response is JSON; the side-by-side normalised view lives at result.normalized.

Health checks: GET /livez, GET /readyz, GET /health. OpenAPI/Swagger UI: /docs.
